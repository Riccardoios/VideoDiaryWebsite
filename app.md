## Feature inventory (from listing + screenshots)

- **One-tap recording** — open the app, tap REC, done. Front/rear camera switch.
- **Calendar view** — every day shows its video thumbnail; scroll months of memories.
- **Gallery view** — all videos with durations; filter All Videos / Montages.
- **Montage maker** — combines daily videos into one movie ("Create Montage", shows progress %).
- **Journaling prompts** — daily reflection question on the record screen (e.g. "What brings you comfort today?").
- **Mood tracking** — attach a mood to each entry; Insight tab for patterns.
- **Notes** — optional written note alongside each video.
- **Multiple videos per day**, customizable reminders.
- **Privacy** — videos stay on device / user's iCloud; no AI processing, no hidden sharing.
- **Premium** — exclusive color palettes (Classic, Modern Red + dark mode, Ocean, Forest, Sunset…), unique fonts, Face ID lock, iCloud backup.

# Keyword research

**Positioning insight:** the winnable angle is *simplicity + privacy + no AI*. Head terms ("journal app") are owned by Day One and listicles; but video-journaling queries have blog-grade SERPs (memovi.app, uly.app, calmsage.com, psychologytoday.com rank with simple articles), which a focused single-product site can crack. Every keyword below has either download intent or a problem the app directly solves.

## SERP competitors observed

- **memovi.app** — blog cluster ("How to Start a Video Journal", "10 Best Video Diary Apps") ranking well; validates the guide-cluster strategy.
- **1 Second Everyday (1se.co)** — owns "1 second a day"; we target it via guide #4 as an alternative angle.
- **My Life Journal, Orca (okorca.com), uly.app, Meminto, Remento** — landing pages + benefit articles.
- **calmsage.com / psychologytoday.com** — rank for mental-health journaling angles with generic articles (guide #7 opportunity).

## On-page SEO checklist applied to the site

- Unique title (≤60 chars) + meta description (≤160 chars) per page, keyword-leading.
- One H1 per page containing the target keyword; H2s cover related subqueries.
- Canonical URLs on `https://videodiary.app`; sitemap.xml + robots.txt.
- JSON-LD: `SoftwareApplication` + `FAQPage` (home), `Article`/`HowTo` + `BreadcrumbList` (guides).
- Open Graph + Twitter cards with og-image.
- Descriptive, keyword-bearing image alt text; explicit width/height; lazy loading below the fold.
- Internal linking: FAQ → guides ("Learn more"), guides cross-link 3 related guides, all guides link home + App Store.
